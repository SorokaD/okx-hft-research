# Перевод research-кода в production

## Критерии готовности

- [ ] Фича/сигнал покрыт тестами
- [ ] Валидация (стабильность, корреляции) выполнена
- [ ] Docstring и типизация
- [ ] Конфиг через env/yaml, без хардкода
- [ ] Документация в `docs/` или в эксперименте
- [ ] Code review

## Целевые репозитории

- **okx-hft-airflow-dags** — ETL для фичей, агрегаты
- **okx-hft-executor** — сигналы, исполнение ордеров

## Процесс

1. Зафиксировать результат эксперимента в `experiments/EXP-NNN-*/results.md`
2. Выделить код в отдельный модуль с чётким API
3. Написать тесты
4. Создать PR в целевой репозиторий с ссылкой на эксперимент
